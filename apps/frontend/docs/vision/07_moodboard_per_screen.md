# 画面別ムードボード指示書

> 最終更新: 2026-04-28
> 出典: [Codex 提案](/tmp/flovia-codex-review/moodboard_per_screen.md)
> 前提: [06_design_direction.md](06_design_direction.md) のハイブリッド方針

## ★ 通底テーマ (4 画面共通)

> **「静かな精度の上に、見えない関係性が浮かび上がる」**

- UI shell は一貫して端正・抑制的、信頼感を担保する器として振る舞う
- 主役となる分析画面だけが、**青いメッシュの気配** で深さと独自性を獲得する
- 色も動きも "派手さ" ではなく "構造の可視化" のために使う
- 結果として、crypto 的熱狂ではなく **機関投資家にも通じる冷静な洗練** を目指す

## ★ Setup / Onboarding (`/setup`)

### 目的
初回利用時に `pay_to` アドレスを迷わせず登録させ、Flovia の分析体験に入る最短導線を作る。

### 第一印象
「難しい設定画面」ではなく、「信頼できるプロダクトの入口」であることを 5 秒で感じさせる。

### レイアウト

```
+----------------------------------------------------------------------------------+
| Sidebar (264)        | Header: Provider selector / Saved pay_to shortcuts       |
|                      +-----------------------------------------------------------+
|                      |   [ Intro block ]                                         |
|                      |   Title + 1 sentence + mode toggle                        |
|                      |                                                           |
|                      |   +------------------------+  +------------------------+  |
|                      |   | Simple Mode            |  | Advanced Mode          |  |
|                      |   | pay_to input           |  | multiple fields / tags |  |
|                      |   | helper text            |  | validation rules       |  |
|                      |   +------------------------+  +------------------------+  |
|                      |                                                           |
|                      |   [ Saved pay_to list ]                                   |
|                      |   compact rows / switch / delete                          |
|                      |                                                           |
|                      |   [ Primary CTA ]                                         |
+----------------------------------------------------------------------------------+
```

### 視覚的強弱
- 主役 = 登録フォーム (画面中央、最も明るい面)
- モード切替 = フォーム直上、操作の起点として強め
- 保存済み一覧 = 1 段弱いコントラスト
- 補助説明 = 最小限。説明で埋めず、入力のしやすさで安心感を作る

### タイポグラフィ
| 要素 | 値 |
|---|---|
| 見出し | 32px / 700 / Geist Sans |
| セクション見出し | 18px / 600 |
| 入力値 | 15px / 500 / Geist Mono 併用可 |
| 本文 | 14px / 400 |
| ラベル | 12px / 500 / letter-spacing やや広め |
| CTA | 14px / 600 |

### 配色
- 背景: `#020617` 基調、カード面 `rgba(15,23,42,0.72)` で重ねる
- Teal `#2DD4BF`: CTA / フォーカスリング / 入力完了状態に限定
- Mesh blue `#60A5FA`: モード切替の選択状態と微細な区切り線程度に抑制
- グレーは 4 段階 (背景 / 面 / 主要文字 / 補助文字)

### マイクロインタラクション
| 要素 | 動き |
|---|---|
| モード切替 | 180ms ease-out, 背景ピルがスライド |
| 入力フォーカス | 120ms, border + glow が Teal に遷移 |
| 保存済み行 hover | 140ms, 背景 +3% |
| CTA 活性化 | 160ms, opacity → solid |

### 状態
- ローディング: CTA 内インジケータ中心 (フォーム skeleton は不要)
- 空: 「まだ何も登録されていない」と言うだけ。イラスト不要
- エラー: 入力欄直下に短くインライン表示。上部の赤バナーは禁止

### リッチに見える 3 つ
1. フォームカード背面に **Teal と Blue の二層グロー** (ごく弱く)
2. 保存済み `pay_to` 行に mono 省略表示 + subtle copy affordance
3. モード切替時に **クロスフェード + 高さ補間** (即切替ではない)

### 避けるべきこと
- web3 wallet 接続画面のような派手な発光ボタン
- 説明文の長文化
- Advanced モードを「開発者向けの罰ゲーム」に見せること
- 赤・緑依存の状態表現

## ★ My Customers (`/providers/[providerId]/customers`)

### 目的
Provider 配下の顧客ウォレットを一覧で把握し、次に深掘る対象を素早く選ばせる。

### 第一印象
「整理されている」「信頼できる」「スキャンしやすい」を即座に感じさせる。

### レイアウト

```
+----------------------------------------------------------------------------------+
| Sidebar              | Header: Provider selector / search / filters             |
|                      +-----------------------------------------------------------+
|                      | [ Page title ]      [ summary chips ]                    |
|                      |                                                           |
|                      | +------------------------------------------------------+  |
|                      | | Toolbar: search | segment | sort | export?          |  |
|                      | +------------------------------------------------------+  |
|                      |                                                           |
|                      | +------------------------------------------------------+  |
|                      | | Customers table                                       |  |
|                      | | Address | Activity | Last seen | Score | Status ...   |  |
|                      | |------------------------------------------------------|  |
|                      | | row                                                > |  |
|                      | +------------------------------------------------------+  |
|                      |                                                           |
|                      | [ Pagination / result count ]                            |
+----------------------------------------------------------------------------------+
```

### 視覚的強弱
- 主役 = テーブルそのもの。KPI カードで競合させない
- ツールバー = 操作起点として明確だが、主役化しない
- summary chips = 補助情報。視線の初動だけ支える
- 行 hover / 選択状態で「次のクリック先」を示す

### タイポグラフィ
| 要素 | 値 |
|---|---|
| ページ見出し | 24px / 700 |
| サマリー数値 | 18px / 600 / Geist Mono |
| テーブル本文 | 14px / 400 |
| アドレス表示 | 13px / 500 / Geist Mono |
| ヘッダーラベル | 12px / 600 |
| 行内補助情報 | 12px / 400 |

### 配色
- **Precision Graph 優先**。Mesh blue は極小使用
- Teal: 選択行 / ソート active / 検索フォーカス / 重要ステータスのみ
- テーブル区切り = 高コントラスト線ではなく半透明細線
- 行 hover = 背景面 +4-6%
- 危険系 = desaturated red を最小限

### マイクロインタラクション
| 要素 | 動き |
|---|---|
| 行 hover | 100ms, 背景変化 + 右矢印 4px 移動 |
| ソート切替 | 120ms, caret 回転 |
| 検索入力 | 140ms, フォーカスリング |
| テーブル初期表示 | 220ms, 行が下から 8px フェードイン |

### 状態
- ローディング: table skeleton (列幅を本番に近づける)
- 空: 「対象顧客なし」ではなく **次の行動を明示**
- フィルタ結果ゼロ vs データ未登録ゼロを **別メッセージ**
- エラー: テーブル領域内インライン表示、全画面遮断しない

### リッチに見える 3 つ
1. 行 hover 時、address の mono テキストだけが **少し明るくなる**
2. summary chips に **微細な縦グラデ + 薄い内側ハイライト**
3. 選択行の左端に **2px の Teal インジケータ**

### 避けるべきこと
- KPI カードを上に並べすぎてテーブルの主役感を失う
- バッジ色の乱用
- 行ごとにアイコン多用で情報密度を下げる
- crypto ダッシュボード風の蛍光色テーブル

## ★ Wallet 360° (`/providers/[providerId]/wallet/[address]`)

### 目的
単一ウォレットの行動・関係性・示唆を、最も印象的かつ説得力ある形で提示する **主役画面**。

### 第一印象
「これはただの管理画面ではなく、構造を見せる分析プロダクトだ」と感じさせる。

### レイアウト

```
+----------------------------------------------------------------------------------+
| Sidebar              | Header: Provider / wallet breadcrumb / actions           |
|                      +-----------------------------------------------------------+
|                      | [ Wallet identity bar ]                                   |
|                      | address / labels / risk? / key stats                      |
|                      |                                                           |
|                      | +----------------------------------+--------------------+ |
|                      | | Activity Timeline                | Insight stack      | |
|                      | |                                  | card 1             | |
|                      | |                                  | card 2             | |
|                      | |                                  | card 3             | |
|                      | +----------------------------------+--------------------+ |
|                      |                                                           |
|                      | +------------------------------------------------------+  |
|                      | | Co-usage Map (hero network visualization)            |  |
|                      | |                    node graph                         |  |
|                      | +------------------------------------------------------+  |
|                      |                                                           |
|                      | [ supporting detail strip / legend / filters ]           |
+----------------------------------------------------------------------------------+
```

**主役論点は解決済み (D14)**: デモ脚本 ([08_demo_script.md](08_demo_script.md)) のクライマックス a は Timeline でしか成立しないため、**Activity Timeline を主役で固定**。Co-usage Map は補強役 (画面下半分の視覚的存在感は維持しつつ、序列上は 4 番目)。詳細: [10_design_review.md](10_design_review.md)。

### 視覚的強弱
- 主役 = Co-usage Map (画面下半分を広く使い、視覚的支配力)
- 次点 = Timeline (因果や推移を理解させる analytical anchor)
- Insight cards = 右側に積む。「結論の要約」に徹する (説明過多にしない)
- Identity bar = 信用の土台。派手さより端正さ優先

### タイポグラフィ
| 要素 | 値 |
|---|---|
| ページ / Wallet 主見出し | 28px / 700 / **Space Grotesk 可** |
| セクション見出し | 18px / 600 |
| 大きな数値 / KPI | 24px / 600 / Geist Mono |
| 本文 | 14px / 400 |
| ラベル / 凡例 | 12px / 500 |
| アドレス / ID | 13px / 500 / Geist Mono |

### 配色
- 背景 `#0A0F1E` (UI shell より奥行きを出す)
- Co-usage Map の主光源 = **Mesh blue `#60A5FA`** (ネットワーク表現は青主体)
- Teal = 選択ノード / 重要 insight / 相互作用 affordance に限定
- グラス面 `rgba(10,15,30,0.58)` 前後、境界に薄い白ハイライト
- グレースケールは黒寄りで締め、白を使いすぎない

### マイクロインタラクション
| 要素 | 動き |
|---|---|
| Map ノード出現 | 280ms stagger, opacity + scale 0.96→1 |
| ノード hover | 120ms, outer glow + 接続線強調 |
| Timeline scrub/hover | 100ms, カーソル追従の縦ガイド |
| Insight card update | 180ms, content crossfade |
| 画面ロード時 | hero map 背景メッシュが **8〜10 秒周期でごく弱く揺らぐ** |

### 状態
- ローディング: hero visualization の **構図を保った skeleton**
- Map データ待ち時も背景グリッドや faint node placeholders を出し、空白を作らない
- 空: 「関係がない」ではなく **「十分な co-usage signal が未検出」** と表現
- エラー: 可視化領域内に原因と再試行を出し、その他セクションは残す

### リッチに見える 3 つ
1. Co-usage Map 背面に非常に淡いブルーの **volumetric haze**
2. 選択ノードに連動して **Timeline 上の該当期間が同期ハイライト**
3. Insight card 先頭に 1 行だけ「machine summary」風の **鋭い短文**

### 避けるべきこと
- ネオン過多でサイバー系に寄ること
- ノード/線が多すぎて意味のない "複雑さの演出" になること
- Insight を長文カードにすること
- すべてを青発光で統一して情報階層を失うこと

## ★ Co-usage Patterns (`/providers/[providerId]/patterns`)

### 目的
個別ウォレットの理解を超えて、Provider 全体の共通パターンとクラスタ傾向を **俯瞰** させる。

### 第一印象
「全体構造が見える」「分析に再現性がある」と感じさせる。

### レイアウト

```
+----------------------------------------------------------------------------------+
| Sidebar              | Header: Provider / timeframe / filters                   |
|                      +-----------------------------------------------------------+
|                      | [ Title + compact explanation ]                           |
|                      |                                                           |
|                      | +-------------------------------+----------------------+  |
|                      | | Bubble Chart                  | Retention by Agent   |  |
|                      | | dominant visual               | compact comparison   |  |
|                      | +-------------------------------+----------------------+  |
|                      |                                                           |
|                      | +------------------------------------------------------+  |
|                      | | Workflow Clusters                                      | |
|                      | | cluster lanes / cards / step groupings                 | |
|                      | +------------------------------------------------------+  |
|                      |                                                           |
|                      | [ legend / notes / drilldown controls ]                  |
+----------------------------------------------------------------------------------+
```

### 視覚的強弱
- 主役 = Bubble Chart (最初の理解を一気に作る俯瞰装置)
- 次点 = Workflow Clusters (構造の意味を言語化する分析面)
- Retention by Agent = 比較指標として右上、補助だが鋭く
- 凡例 = 必要最小限、解釈を邪魔しない位置に退避

### タイポグラフィ
| 要素 | 値 |
|---|---|
| ページ見出し | 24px / 700 |
| 可視化注記 | 12px / 400 |
| 指標数値 | 20px / 600 / Geist Mono |
| クラスタ名 | 16px / 600 |
| 本文 | 14px / 400 |
| 軸ラベル / 凡例 | 11〜12px / 500 |

### 配色
- Bubble Chart は **neutral base** 上で、主要クラスタに Teal / Mesh blue を核にした **限定パレット**
- 主系列 = Teal、関係系列 / 密度表現 = Mesh blue、その他 = desaturated slate/cyan
- Workflow Clusters カード背景 = shell 系ダーク、強調線に Teal
- Retention = 単純な棒色ベタ塗りではなく **低彩度グラデーション**

### マイクロインタラクション
| 要素 | 動き |
|---|---|
| Bubble 初期配置 | 300ms, 位置補間 + soft scale-in |
| Bubble hover | 100ms, 輪郭強調 + tooltip フェード |
| Cluster selection | 160ms, 関連要素を残し他を 35% 減光 |
| Retention chart bars | 220ms, 下から grow |
| Workflow lane expand/collapse | 180ms, 高さ補間 |

### 状態
- ローディング: 可視化の **骨格を先に表示** (軸 / 枠 / プレースホルダ円)
- 空: 期間条件が厳しいのか、データ自体が薄いのかを **分ける**
- エラー: 画面全体停止ではなく、**各 visualization 単位で recoverable** に
- 凡例 / フィルタは残し、ユーザーが次の行動を取れるように

### リッチに見える 3 つ
1. Bubble の **重なり部分に薄い additive glow** (密度感)
2. Cluster 選択時、下段 Workflow Clusters が **連動ハイライト**
3. Retention chart の現在注目系列だけ **subtle outer line** を追加

### 避けるべきこと
- 可視化ごとに別デザイン言語になって統一感を失う
- カラフルにしすぎて "分析玩具" に見える
- 軸 / 凡例が小さすぎて読めない
- 集約ビューなのに情報を詰め込みすぎて俯瞰性を壊す
