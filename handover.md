## Context

This repository is the migration target for Flovia 260427 work from `foxytanuki`.

```text
Relocate now. Refactor later.
```

- `payment_observations` as source-of-truth facts
- `settlement_evidence`

- Daily metrics
- Payer wallet profiles
- Recipient summaries
- Relayer summaries

## Important invariant

```text
Onchain observations are facts.
Catalog and fingerprint data only enrich observed facts.
Provider and middleman labels stay candidates with confidence and evidence.
```

Do not put provider or middleman final labels into `payment_observations`.

## Near-term phases

### Phase 1: Keep CLI baseline green

- Preserve `apps/cli` behavior.
- Keep `bun run verify` offline.
- Keep fixture tests deterministic.

### Phase 2: Add RPC tx ingest inside `apps/cli`

Add a script like:

```text
apps/cli/scripts/ingest-rpc-tx.ts
```

It should:

- accept a tx hash
- use `BASE_RPC_URL`
- fetch `eth_getTransactionByHash`
- fetch `eth_getTransactionReceipt`
- normalize into current raw tx / receipt types
- reuse the existing decoder and observation builder
- insert observations idempotently

Offline verify must remain RPC-free.

### Phase 3: Add bounded range ingest inside `apps/cli`

Add a script like:

```text
apps/cli/scripts/ingest-rpc-range.ts
```

Candidate filters:

```text
tx.to == Base USDC and selector == transferWithAuthorization
tx.to == Multicall3 and selector == aggregate3
```

Fetch receipts only for candidates.

### Phase 4: Add indexer foundation

Add cursor and run metadata only after tx and range ingest work.

Likely additions:

- `ingestion_runs`
- `sync_cursors`
- raw transaction / raw log audit tables
- parser version
- rule version
- finality delay
- retry and rate-limit handling
- reorg detection via block hash checks

### Phase 5: Extract packages from proven seams

Only extract packages after online ingest creates real duplication or a second consumer exists.

Expected eventual packages:

```text
packages/core
packages/db
packages/collector
packages/report
```

`apps/cli` should become a thin command layer over those packages.

### Phase 6: Add API and dashboard later

Later apps:

```text
apps/api
apps/dashboard
```

`apps/api` is a read-only BFF for dashboard projections and evidence drill-downs.

## Files intentionally ignored

- `.env`
- local SQLite DB files
- `node_modules`
- generated report summaries under `apps/*/reports/summary.*`

Reports are generated by `bun run verify` or `bun run report`.

## Do not do yet

- Do not build the UI first.
- Do not make default verify depend on live RPC.
- Do not split packages before RPC tx / range ingest clarifies boundaries.
- Do not collapse attribution candidates into final labels.
